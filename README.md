## LR‑Chip — безопасный статический сайт для Netlify

Этот сайт спроектирован как «по умолчанию безопасный»: полностью статический, без клиентских секретов и сторонних скриптов, со строгими HTTP‑заголовками и минимальным фронтендом.

### Что внутри
- Страницы: `index.html`, `thank-you.html`, `404.html`
- Статика: `styles.css`, `script.js`, `robots.txt`
- Конфигурация: `netlify.toml`, `_headers` (дублирование на случай разных способов конфигурации)
- Форма обратной связи: Netlify Forms с honeypot, без внешних зависимостей

### Безопасность по умолчанию
- Строгий CSP: `default-src 'self'`; запрет `object-src`; запрет встраивания (`frame-ancestors 'none'`);
  из внешнего разрешены только собственные ресурсы. Нет inline‑скриптов/стилей и event‑handlers.
- HSTS: принудительный HTTPS (+ `preload`).
- `X-Frame-Options: DENY`, `X-Content-Type-Options: nosniff`, `Referrer-Policy: no-referrer`.
- `Permissions-Policy`: отключены потенциально опасные браузерные API.
- COOP/COEP/CORP: строгая изоляция контента (не подключайте внешние ресурсы без корректных CORS/Corp).

Важно: 100% неуязвимых сайтов не бывает. Здесь мы минимизируем поверхность атаки: нет сервера/БД, нет стороннего кода, строгие заголовки и CSP.

### Деплой на Netlify
1. Создайте репозиторий (GitHub/GitLab/Bitbucket) и загрузите файлы в корень.
2. В Netlify нажмите «New site from Git», выберите репозиторий.
3. Build command оставьте пустым, `Publish directory` — `.` (корень).
4. Задеплойте. Netlify автоматически выпустит TLS‑сертификат.

### Netlify Forms
- Форма уже настроена: `data-netlify="true"` + `netlify-honeypot`.
- По умолчанию пост будет перехвачен Netlify и отрендерит `thank-you.html`.
- В админке Netlify включите уведомления (email/Slack) при новых заявках.

### Рекомендации при доработках
- Не добавляйте inline‑скрипты/стили — нарушит CSP.
- При подключении внешних ресурсов обновляйте CSP и, при необходимости, снимайте COEP/Corp или добавляйте заголовки на стороне источника.
- Не размещайте в репозитории ключи/секреты — используйте Netlify Environment Variables и серверные функции (если они появятся).
